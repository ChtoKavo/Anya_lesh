# Документация: Админ Панель и Расширенные Функции Профиля

## Обзор внедренных функций

### 1. **Админ Панель** 🔐
- Управление всеми пользователями системы
- Просмотр информации о пользователях, их питомцах и прогрессе
- Система рейтинга с топ пользователями
- Управление запросами в поддержку

**Маршрут:** `/admin`
**Доступ:** Только для пользователей с ролью `admin`

#### Функции админ панели:
- **Пользователи** - Полная информация о всех пользователях с поиском и фильтрацией
- **Рейтинг** - Топ пользователей по XP с возможностью развернуть полный список
- **Поддержка** - Управление запросами в поддержку (просмотр, ответ, статус)
- **Статистика** - Общая статистика системы (кол-во пользователей, питомцев, запросов)

### 2. **Расширенная Профильная Страница** 👤
Новые вкладки в профиле пользователя:

#### Редактирование профиля
- Изменение имени
- Изменение email
- Изменение никнейма персонажа
- Изменение аватара

#### Безопасность
- Изменение пароля с проверкой текущего пароля
- Требование подтверждения нового пароля

#### Поддержка
- Кнопка "Написать поддержку"
- Модальное окно с формой для отправки запроса
- История всех запросов в поддержку с их статусами
- Просмотр ответов от администратора

### 3. **Система Поддержки** 💬
- Каждый пользователь может отправить запрос в поддержку
- Выбор категории (Баг, Предложение функции, Проблема с аккаунтом, Другое)
- Статусы: open, in_progress, resolved, closed
- Администратор может ответить на запрос

### 4. **Система Рейтинга** 🏆
- Автоматический рейтинг пользователей по XP
- Отображение питомца и уровня рядом с пользователем
- Развертываемый список для просмотра всех пользователей

## Установка и Миграция БД

### 1. Применить миграцию БД

Выполнить SQL команды из файла `database/migration_admin_features.sql`:

```bash
# Подключиться к MySQL
mysql -u root -p mydiplom

# Выполнить миграцию
source /path/to/database/migration_admin_features.sql;
```

Или выполнить вручную в MySQL Workbench/PhpMyAdmin:

```sql
-- Скопировать и выполнить весь контент из migration_admin_features.sql
```

### 2. Создать админ пользователя (опционально)

Для тестирования можно создать админа через скрипт или вручную в БД:

```sql
-- Вручную обновить роль существующего пользователя
UPDATE users SET role = 'admin' WHERE email = 'your_email@example.com';
```

### 3. Перезагрузить backend сервер

```bash
cd mydiplom-backend
npm start
```

### 4. Перезагрузить frontend

```bash
cd mydiplom
npm run dev
```

## Структура Backend

### Новые контроллеры:
- `src/controllers/users.controller.js` - Управление пользователями для админов
- `src/controllers/support.controller.js` - Управление поддержкой

### Новые маршруты:
- `src/routes/admin.routes.js` - API для админ панели
- `src/routes/support.routes.js` - API для поддержки

### Новое middleware:
- `src/middleware/admin.js` - Проверка админ статуса

### Расширенные контроллеры:
- `src/controllers/profile.controller.js` - Новые функции редактирования

### API Endpoints:

#### Админ панель:
- `GET /api/admin/stats` - Статистика системы
- `GET /api/admin/all` - Все пользователи
- `GET /api/admin/user/:id` - Информация о пользователе
- `GET /api/admin/ranking` - Рейтинг пользователей
- `DELETE /api/admin/user/:id` - Деактивировать пользователя
- `POST /api/admin/user/role` - Изменить роль пользователя

#### Поддержка:
- `POST /api/support/request` - Отправить запрос в поддержку
- `GET /api/support/requests/:userId` - Получить запросы пользователя
- `GET /api/support/all` - Все запросы (только админ)
- `POST /api/support/respond` - Ответить на запрос (только админ)
- `PUT /api/support/close/:id` - Закрыть запрос (только админ)

#### Профиль:
- `PUT /api/profile/update` - Обновить профиль
- `POST /api/profile/change-password` - Изменить пароль
- `PUT /api/profile/pet/:petId` - Обновить питомца

## Как пользователь может написать в поддержку?

1. Перейти в **Профиль** → вкладка **Поддержка**
2. Нажать кнопку **"📧 Написать поддержку"**
3. Заполнить форму:
   - Тема запроса
   - Категория (Баг, Предложение, Проблема с аккаунтом, Другое)
   - Сообщение с подробным описанием
4. Нажать **"Отправить"**
5. Запрос появится в истории с статусом "open"
6. Админ может ответить на запрос
7. Пользователь увидит ответ в профиле

## Как администратор управляет системой?

1. Перейти на **Админ Панель** (ссылка в хэдере, если роль = admin)
2. **Вкладка "Пользователи"**:
   - Видеть список всех пользователей
   - Искать по email или нику
   - Просмотреть подробную информацию
   
3. **Вкладка "Рейтинг"**:
   - Видеть топ пользователей
   - Просмотреть их питомцев и уровни
   - Развернуть полный список

4. **Вкладка "Поддержка"**:
   - Фильтровать по статусу
   - Просмотреть запросы пользователей
   - Ответить на запрос
   - Изменить статус

5. **Вкладка "Статистика"**:
   - Видеть общую статистику
   - Топ 5 активных пользователей

## Информация в профиле пользователя

### Что может редактировать пользователь?
1. **Имя** (реальное имя)
2. **Email** (с проверкой уникальности)
3. **Никнейм персонажа** (имя персонажа)
4. **Аватар** (буква из имени)
5. **Пароль** (с проверкой текущего пароля)

### Информация в админ панели о пользователе
1. Email
2. Имя
3. Никнейм
4. Питомец (имя, тип, уровень, XP, настроение)
5. Уровень пользователя
6. Количество изученных слов
7. Дни подряд (streak)
8. Монеты
9. Дата регистрации
10. Статус (активный/неактивный)

### Информация о питомце в админ панели
1. Имя питомца
2. Тип питомца
3. Уровень
4. XP
5. Настроение
6. Рост/стадия развития

## Система Рейтинга

### Как рассчитывается рейтинг?
1. **Первичная сортировка:** По XP (основной параметр)
2. **Вторичная сортировка:** По уровню
3. **Третичная сортировка:** По количеству монет

### Что показывается в рейтинге?
- Место в рейтинге (#1, #2 и т.д.)
- Никнейм пользователя
- Имя питомца и его тип
- Уровень пользователя
- Количество XP
- Количество монет
- Количество выученных слов

## Безопасность

### Проверки для админа
- Только пользователи с ролью `admin` могут видеть админ панель
- При попытке доступа как обычный пользователь - редирект на профиль
- Все админ API endpoints защищены middleware проверкой

### Проверки для профиля
- Изменение email - проверка уникальности
- Изменение пароля - требуется текущий пароль
- Система поддержки - только авторизованные пользователи
